## Release highlights

=== "🧾 Software Bill of Materials (SBOM)"
    ### SBOM's for PCSM

    Starting with Percona ClusterSync for MongoDB (PCSM) 0.9.0, we now provide Software Bills of Materials (SBOMs) for each release. SBOMs improve software supply chain transparency by documenting the components and dependencies included in a build.

    PCSM SBOMs are generated in the industry-standard `CycloneDX 1.6` format and are automatically produced as part of the release pipeline.

    To ensure easy access, SBOMs will be distributed through multiple channels for PCSM:

    === ":octocat: GitHub" 
        The latest SBOM will be placed in the root of the main branch.

    === "📦 Binaries/Tarballs" 
        An SBOM file will be included in the root directory of the release archive.

    === "📜 Packages (`.deb/.rpm`)" 
        The SBOM file will be placed in a standard documentation directory, such as `/usr/share/doc/`.

    === "🐳 Docker" 
        The SBOM will be attached directly to the Docker image as metadata.

    === "🏛️ Percona Trust Center"
         We will investigate automating the upload of SBOMs to the Trust Center to maintain a centralized, up-to-date repository.

### Improvements

- [PCSM-176](https://perconadev.atlassian.net/browse/PCSM-176): PCSM 0.9.0 enhances visibility into the progress and outcomes of the replication finalization process. You can now monitor the state of the PCSM finalize operation in real time using the `/status` endpoint. While finalization is underway, the `/status` reports that finalization is running. Once the process is complete, this endpoint provides a structured summary of the finalization results, including details about any **unsuccessful index builds**. This allows you to identify and diagnose index-related issues without the need to manually inspect logs.
