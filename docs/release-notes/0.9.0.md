## Release highlights

=== "🧾 Software Bill of Materials (SBOM)"
    ### SBOM's for PCSM

    Starting with Percona ClusterSync for MongoDB (PCSM) 0.9.0, we now provide Software Bills of Materials (SBOMs) for each release. SBOMs improve software supply chain transparency by documenting the components and dependencies included in a build.

    PCSM SBOMs are generated in the industry-standard `CycloneDX 1.6` format and are automatically produced as part of the release pipeline.

    To ensure easy access, SBOMs will be distributed through multiple channels for PCSM:

    === "<img src="https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png" alt="GitHub" width="20" height="20"> **[GitHub](ca://s?q=PCSM_SBOM_on_GitHub)**" 
        The latest SBOM will be placed in the root of the main branch.

    === "📦 Binaries/Tarballs" 
        An SBOM file will be included in the root directory of the release archive.

    === "📜 Packages" 
        For `.deb` and `.rpm` packages, the SBOM file is installed in a standard documentation directory such as `/usr/share/doc/`.

    === "🐳 Docker" 
        The SBOM will be attached directly to the Docker image as metadata.

    === "🏛️ Percona Trust Center"
         We will investigate automating the upload of SBOMs to the Trust Center to maintain a centralized, up-to-date repository.

=== "Cross-version replication"
    ### Replication from 6.0 ➡️ 8.0 and 7.0 ➡️ 8.0

    Percona ClusterSync for MongoDB (PCSM) now supports replication only for MongoDB major version combinations. In the current release, this includes clusters running the same major version and selected upgrade paths from lower to higher. This lets you synchronize data during a staged upgrade or maintain replication across clusters running different versions, provided support is explicitly enabled. 

    When PCSM starts, it checks the major version relationship between the source and target clusters. If the source major version is higher than the target major version, PCSM will block startup and exit with an error.

    For detailed information on this feature, see our [documentation]().
    


## Changelog

## New features

- [PCSM-282](https://perconadev.atlassian.net/browse/PCSM-282) Starting with Percona ClusterSync for MongoDB (PCSM) 0.9.0, we now provide Software Bills of Materials (SBOMs) for each release.

- [PCSM-286](https://perconadev.atlassian.net/browse/PCSM-286): PCSM 0.9.0 adds support for cross-version replication from MongoDB 7.0 to MongoDB 8.0.

- [PCSM-289](https://perconadev.atlassian.net/browse/PCSM-289): PCSM 0.9.0 adds support for cross-version replication from MongoDB 6.0 to MongoDB 8.0.

### Improvements

- [PCSM-176](https://perconadev.atlassian.net/browse/PCSM-176): PCSM 0.9.0 enhances visibility into the progress and outcomes of the replication finalization process. You can now monitor the state of the PCSM finalize operation in real time using the `/status` endpoint. While finalization is underway, the `/status` reports that finalization is running. Once the process is complete, this endpoint provides a structured summary of the finalization results, including details about any **unsuccessful index builds**. This allows you to identify and diagnose index-related issues without the need to manually inspect logs.


### Bugs

- [PCSM-249](https://perconadev.atlassian.net/browse/PCSM-249): PCSM now transparently handles `movePrimary`, detecting phantom change stream events produced by it and skipping those events. No operation is performed on the target, whereas previously, this caused a data mismatch resulting in zero documents on the target.

- [PCSM-305](https://perconadev.atlassian.net/browse/PCSM-305):
During heavy replication workloads, pcsm could cause the destination MongoDB instance to crash with a memory limit error (`BufBuilder attempted to grow() to 131072004 bytes, past the 125MB limit`). This resulted in write contention, operation timeouts, and stuck migrations, most notably when migrating large collections from Atlas.

- [PCSM-314](https://perconadev.atlassian.net/browse/PCSM-314) Fixed an issue where pcsm could crash during replication when processing an event against an item that did not exist in the target array, causing the replication to halt unexpectedly.
